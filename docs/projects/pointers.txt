Pointers Project


Description:
    You will implement features that enable better handling of pointers by the W Language.
    Currently, char* and int are two of the main types in use by the compiler.
    However they are broken in different ways involving the &, *, [] operators.
    We need to correct the handling of these operators for char, int16, int32, struct.

Current Status:
    Complete. All milestones are implemented and covered by tests/pointer_test.w:
    char* indexing and dereference, int* indexing, local &, struct pointer fields,
    arrays of structs through pointer indexing, struct values as by-value
    parameters (stack copy with mutation isolation), constructor-style
    new point(1, 2, 3), pointer/type compatibility warnings (make warning_test),
    and multi-level pointer regression coverage (int**, char**).



Specific Scenarios:
    value = *char_ptr
    *char_ptr = value
    
    value = int_ptr[3]
    int_ptr[4] = value

    field_value = struct_ptr.field_name
    struct_ptr.field_name = new_value

    field_value = struct_list[5].field
    struct_list[6].field_value = new_value
