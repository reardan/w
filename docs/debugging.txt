Current state:

	the compiler emits DWARF line-number information for gdb
	the debugger statement emits int3
	repl.w provides an in-process interactive prompt
	repl compile errors return to the prompt

	wdbg (debugger/debugger.w, built with make wdbg) is the built-in debugger:

		./bin/wdbg file.w

	compiles file.w into an in-process executable buffer (the repl model),
	installs a SIGTRAP handler via rt_sigaction, and calls the debuggee's
	main(). Every 'debugger' statement traps into a command loop:

		c / continue    resume the debuggee
		r / registers   dump the trapped registers from the signal frame
		s / stack       dump 16 words at the trapped esp
		l / line        map the trapped eip to file:line via the DWARF notes
		q / quit        exit

		./bin/wdbg file.w --break_start

	traps before the debuggee's main runs

		./bin/wdbg file.w --break_end

	traps after the debuggee's main returns

	make debug_test drives the command loop over stdin against
	tests/debug_fixture.w and asserts each command's output.
