current state:

	build and verification...
		working:
			self-hosted x86 bootstrap:
				./w w.w -> bin/wv2 -> bin/wv3 -> bin/wv4 -> bin/wv5
			make verify:
				compares wv3 == wv4 == wv5
			make tests:
				runs compiler, grammar, lib, pointer, struct, range,
				data-structure, import, directory, threading, hello tests
			make update:
				archives the current seed and promotes bin/wv2 to ./w
			compiler CLI:
				-o output path
				usage message instead of segfaulting when no input is passed

	language and compiler features working:
		pointers:
			int*
			char*
			char* indexing
			char* dereference and assignment
			int* indexing
			& local address
			struct pointer fields
			arrays of structs via pointer indexing
		type table:
			type names carry size, pointer level, and field metadata
			pointer types are created on demand
			struct fields store field type and offset information
			int8, int16, int32 and unsigned integer types are registered
			types_compatible() checks pointer depth, base type and struct identity
			assignment, initialization, call-argument and return mismatches warn
			function symbols record parameter types for call-site checks
			make warning_test asserts warning output against fixtures
		whitespace hygiene:
			warning when a line is indented with spaces instead of tabs
			warning when a file does not end with a newline
		struct:
			local struct values
			mixed-width struct fields
			struct pointers
			struct arrays
			new point() and new point allocate struct storage
			new point(1, 2, 3) initializes fields in declaration order
			struct values pass by value (stack copy) as function parameters
		expressions:
			&& and || short-circuit expressions
			! unary operator
			unary - and unary +
			unary operators bind tighter than binary and can stack (!!x, -*p, *&x)
			relational operators chain left-associatively
			hex literals
			string escape handling
			pointer-aware [] and . code generation
		control flow:
			for int i in range N
			range(end), range(start, end), range(start, end, step)
			range arguments can be expressions
			break and continue
			nested if/loop behavior covered by tests
			if/while conditions work with or without parentheses
			{} blocks, dedent-terminated : blocks, and pass as a no-op
		functions:
			function arity warnings
			function pointer values can be called
			function pointer arguments are covered in grammar tests
		debugging information:
			debugger statement emits int3
			DWARF line-number data is emitted for gdb source-line mapping
			wdbg in-process debugger (make wdbg, make debug_test)
		library and data structures:
			malloc/free/realloc as a library module (lib/memory.w)
			list
			hash_map
			string builder
			array_list
			linked_list
			format helpers
			command-line argument parsing (lib/args.w)
		repl:
			make repl builds and runs repl.w
			make repl_test verifies REPL interaction and error recovery
			lines compile into an executable mmap buffer and run immediately
			compile errors roll back state and return to the prompt

	current limitations:
		repl:
			one line per entry
			local variables do not persist between entries
		pointers and structs:
			new initializes fields from arguments but does not run user constructors
			struct values cannot be returned by value
		debugging:
			wdbg breaks only at 'debugger' statements (plus --break_start/--break_end)
			no step/next, named breakpoints, or local inspection yet
			no segv handler or stack parser yet
		x64:
			threading, sockets and the repl/wdbg stubs are still x86-only
			DWARF line info still declares address_size 4 (gdb only)
		webassembly:
			no backend implementation yet
