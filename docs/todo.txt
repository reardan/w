	current limitations:
		repl:
			calls compiled before a redefinition keep the old binding
				(needs a call-site indirection table; tracked in a
				follow-up issue, out of scope for a REPL-only fix)
		pointers and structs:
			new initializes fields from arguments but does not run user constructors
			struct values with fixed-array fields are copied and returned by value
		debugging:
			backtrace is heuristic (return-address scan, no frame pointers),
				so an older frame's base can be unknown (locals unaddressable)
			watchpoints are software (statement-granularity single-step
				scan): large slowdown while any exist
		x64:
			DWARF line info still declares address_size 4 (gdb only)
		threading:
			lib/thread.w (spawn/join/parallel_for) is Linux x86/x64 only;
				arm64/darwin/win64/wasm, mutexes/atomics and thread
				pools are staged in docs/projects/threads.md
			worker stacks (4MB) and handles are not reclaimed on join
		webassembly:
			json builtins, generators, c_import/extern FFI, REPL and wdbg
				are not supported on the wasm target
				(docs/projects/wasm_backend.md)
