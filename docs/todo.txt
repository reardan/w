		whitespace hygiene:

		struct:

		expressions:
			&& and || short-circuit expressions
			! unary operator

			relational operators chain left-associatively
			hex literals
			string escape handling
			pointer-aware [] and . code generation
		control flow:
			for int i in range N

			break and continue

			{} blocks, dedent-terminated : blocks, and pass as a no-op
		functions:
			function arity warnings

		dynamic linking (both targets):
			c_lib "libc.so.6" registers a DT_NEEDED shared library
			extern int getpid() declares an imported C function
			eager GLOB_DAT relocations fill inline GOT slots at load time
			generated shims convert W's stack convention to cdecl (x86)

		debugging information:

		library and data structures:

			list

			string builder

			format helpers

		repl:
			make repl builds and runs repl.w
			make repl_test verifies REPL interaction and error recovery
			entries compile into an executable mmap buffer and run immediately
			multi-line entries (functions, structs, control flow), Python-style:
				':' opens a block ended by a blank line, brackets keep reading
			interactive auto-indent (tty only): ':' indents the next line,
				return/break/continue/pass dedent, a blank line dedents one
				level and ends the entry at column 0
			top-level declarations persist between entries (storage in the buffer)
			redefinition shadows the old symbol (Python-style rebinding)
			bare expressions echo their value by compile-time type
			imports and extern/c_lib declarations work at the prompt
			./bin/repl file.w [args...] runs a file, then attaches the prompt
				to its live definitions (--no_main skips running main)
			compile errors roll back state and return to the prompt

	current limitations:
		repl:
			calls compiled before a redefinition keep the old binding
			struct redefinition keeps the first definition (type_lookup
				returns the first match)
			no line editing or history (needs termios raw mode)
		pointers and structs:
			new initializes fields from arguments but does not run user constructors
			struct values cannot be returned by value
		debugging:
			locals are not visible inside print's compiled expressions
			backtrace is heuristic (return-address scan, no frame pointers)
			no watchpoints or conditional breakpoints yet
		x64:
			threading, sockets and the repl/wdbg stubs are still x86-only
			DWARF line info still declares address_size 4 (gdb only)
		webassembly:
			no backend implementation yet
