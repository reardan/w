# x86-64 instruction corpus harvested from code_generator/ emit() calls and their assembly comments.
# Format: hexbytes|intel-assembly-text. One instruction per line where boundaries are certain;
# multi-instruction sequences use ' ; ' separators in the text field.
#
# Conventions:
# - lowercase hex bytes, no spaces; Intel operand order; [reg+disp] memory form; 0x hex immediates.
# - Relative branches/calls (jmp, je, jne, jns, jl, jle, call rel32) show the dot-relative
#   target ".+N"/".-N" (N measured from the instruction's first byte), matching the formatter
#   and the x86 corpus convention, not the raw rel field.
# - Helpers in x86.w shared between the x86 and x64 targets are harvested in their x64 form:
#   emit_x64_opcode() prepends the REX.W prefix 0x48 when word_size == 8, so source comments
#   written with 32-bit register names (eax, esp, ...) correspond to 64-bit registers here.
# - Runtime-parameterized immediates/displacements are instantiated with the example value from
#   the source comment (op(0x12, 0x345678) = 0x12345678) or a small representative value, and
#   are flagged with a '# parameterized' comment line.
