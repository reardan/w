# x86 (32-bit) instruction corpus harvested from code_generator/ emit() calls and their assembly comments. Format: hexbytes|intel-assembly-text. One instruction per line where boundaries are certain; multi-instruction sequences use ' ; ' separators in the text field.
#
# Provenance: code_generator/x86.w, code_generator/x86_asm.w, code_generator/ffi.w,
# code_generator/sse.w. integer.w and grammar/string_literal.w emit only data or
# route through the x86.w helpers below (call_relative32/pop_eax), so contribute
# no new instruction bytes.
#
# Conventions:
#  - 64-bit-only forms (REX.W \x48 prefix, movq/cvtsi2sd/btc on rax, x64/win64
#    ABI shims) are EXCLUDED; emit_x64_opcode() is treated as emitting nothing
#    (word_size==4).
#  - Some source comments use AT&T order (src,dst, %reg, $imm); text below is
#    normalized to Intel order (dst,src). This is NOT flagged as a mismatch.
#  - Parameterized helpers (runtime immediate/displacement) are instantiated
#    with an example value; a '# parameterized: ...' line precedes the entry.
